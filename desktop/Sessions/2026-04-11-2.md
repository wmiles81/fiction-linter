## Gotchas learned

- **Plan bugs can slip self-review.** I drafted Phase 1 Task 1.1's test case 3 wrong — the `,"` at the end of line 1 closed the first quote, making the test text contradict its stated intent. The implementer caught it by tracing the parity count in a Node repro and escalated BLOCKED. **Trust implementer escalations.** When a subagent says "this test is mathematically incapable of passing with this fix," it's usually right. My self-review checklist was not enough; running the test myself in my head against the exact text of the plan would have caught it.

- **Vitest 4.x is ESM-first.** `require('vitest')` produces an explicit error: `Vitest cannot be imported in a CommonJS module using require(). Please use "import" instead.` Any plan that writes test files in CJS for Vitest 4.x will hit this. ESM imports for test files regardless of the module under test — even when the subject is CJS (`module.exports`), Vite's interop handles named-export destructure transparently at test time.

- **vitest.config.ts `include` is load-bearing and easy to miss.** Two separate gaps this session:
  - Phase 0 forgot `server.fs.allow: ['..']`, making `/@fs/` loader reject `../src/shared/**` files. Hotfix commit `415ac6a`.
  - Phase 0 didn't include `electron/**` in the include glob, so Phase 5's `electron/aiClient.test.js` would have been invisible. Scope-addition in Phase 5.
  Future plans should bake both into Phase 0 from the start.

- **`@uiw/react-codemirror` mounts its `EditorView` via its own internal `useEffect`, which runs after the parent's effects on first mount.** The parent's `useEffect([issues])` in EditorPanel runs BEFORE `onCreateEditor` is called. `viewRef.current` is null at that moment, so the dispatch is skipped. This is why Phase 3's latent race is latent: the guard `if (!view) return` prevents a crash, but if `issues` is already non-empty at mount and never changes again, those findings never reach the editor. See Priority 1 item #4.

- **CodeMirror 6 + `@uiw/react-codemirror` + jsdom is flaky for `user.type` typing tests.** Phase 3's Task 3.6 replaced the typing-based debounce test with a file-reclick pattern that drives content changes through the React state machine without touching the contenteditable. This is the robust way to exercise debouncing once CodeMirror is in place. If a future test needs to simulate keystrokes into the CM6 editor in jsdom, expect pain.

- **`@uiw/react-codemirror` + editor header `<h2>filename</h2>` makes `getByText(filename)` ambiguous.** The tree row (a `<button>`) and the header (an `<h2>`) both render the same file name. Use `getByRole('button', { name: /filename/ })` to scope to the tree row. Phase 3 subagent caught and fixed this inline.

- **Plan-to-implementation deviations are safer when the subagent flags them and the reviewer verifies them.** Four deviations this session (Phase 0 hotfix, Phase 1 plan amendment, Phase 5 ESM/CJS, Phase 5 vitest include, Phase 6 parameter-passing) all survived spec review because they were documented in the commit trail or the dispatch prompt. Silent deviations would have caused confusion.

- **Do NOT skip reviews even under "continue until done" pressure.** Every phase had both spec and code quality review. Two reviewers caught real issues that would have bitten later: Phase 1's test-text bug (implementer, not reviewer, but same principle), Phase 3's initial-load race, Phase 5's missing request timeout, Phase 6's deferred packaging verification. The two-stage review is the quality floor; don't lower it.

- **When subagents can't run shell commands in their sandbox, they fall back to static analysis.** Phase 4 and Phase 5 reviewers both noted "Bash was restricted — I read the files and traced the logic but could not run `npm test` myself." In those cases I re-ran `npm test` in my own session to confirm before moving on. Don't assume a review's pass is authoritative when the reviewer didn't execute code. Always cross-check test status from the main session.

- **TodoWrite at phase-level is the right granularity for a 60-task plan.** Task-level TodoWrite would have been 60+ items; phase-level was 7. Phases are the natural checkpoint unit because reviewer feedback operates at that granularity.
