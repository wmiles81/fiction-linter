## What changed

### 1. Startup race condition — retry loop in main.js

**Root cause:** `electron .` and `vite` start concurrently; Electron reaches `loadURL` before Vite's dev server is listening, gets `ERR_CONNECTION_REFUSED`, and the renderer stays blank.

**What didn't work:** Switched `package.json` dev script to `bash -c 'until nc -z localhost 5173; do sleep 0.1; done && electron .'`. This failed because `concurrently` spawns a shell subshell that doesn't inherit `node_modules/.bin` in PATH, so `electron` resolves to the wrong binary or fails silently.

**Fix:** Reverted `package.json` to the simple parallel form. Added a JS retry loop (25 attempts × 200ms = 5s budget) inside `createWindow` that catches `ERR_CONNECTION_REFUSED` and waits before retrying — no shell tricks needed.

- [electron/main.js](electron/main.js) — `loadURL` wrapped in `async` IIFE with retry loop
- [package.json](package.json) — reverted to `concurrently "vite" "electron ."`
- [vite.config.ts](vite.config.ts) — added `server.watch.ignored: ['**/release/**', '**/dist/**']` to stop Chromium license HTML files triggering spurious hot-reloads

### 3. Suggested fixes: only returning 1 option

**Root cause:** `extractRewriteOptions` regex was `/gs` — missing the `m` (multiline) flag. Without `m`, `^` only matches the very start of the string (position 0), so only item `1.` was ever captured.

**Fix:** Changed to `/gms`.

- [src/App.jsx](src/App.jsx) — `numberedRegex` flag string `'gs'` → `'gms'`

### 4. Suggested fixes: tooltip closing while options are visible

**Root cause:** `scheduleHideTooltip` was an empty-deps `useCallback` with no awareness of `suggestedFixes` state. Moving the mouse off the underlined word would schedule a hide even while the suggestion panel was open.

**Fix:** Added `suggestedFixesRef` as a ref mirror of `suggestedFixes` state (the ref-mirror pattern, already used for `latestIssuesRef`). `scheduleHideTooltip` returns early if `suggestedFixesRef.current !== null` — tooltip stays alive while suggestions are loaded.

- [src/components/editor/Editor.jsx](src/components/editor/Editor.jsx) — `suggestedFixesRef`, its `useEffect` sync, and early-return guard in `scheduleHideTooltip`

### 5. gdoc import: sign-in window hidden behind main window

**Problem:** Clicking a `.gdoc` file started the auth flow (fetch spinner was visible) but the sign-in window opened behind the main Electron window on macOS — no sign-in prompt was visible to the user, so the import appeared to hang.

**Diagnosis:** Added `[gdoc]` logs to `fs:readGdoc` in main.js. Terminal confirmed 200 OK fetches of 25KB+ HTML — the network + conversion pipeline was working. Root cause was window ordering only.

**Fix:** `authWindow.focus()` after `loadURL` in the `gdoc:auth` handler.

- [electron/main.js](electron/main.js) — `authWindow.focus()` added

### 6. isSettingContentRef stuck-true bug

**Root cause (latent, found in code review):** `isSettingContentRef.current` is set `true` before the `markdownToHtml` promise, and reset `false` only in the `.then()` handler. If the `useEffect` cleanup fires (value changed) and the next effect invocation bails early at `if (value === lastEmittedRef.current) return;` (value cycled back to the last-emitted value), then no code path ever resets the ref. The old promise sees `cancelled = true` and returns early too. Result: `isSettingContentRef.current` stays `true` and all subsequent user keystrokes are silently dropped.

**Fix:** Reset `isSettingContentRef.current = false` in the cleanup function. Safe because if the new effect invocation needs async work, it immediately sets the ref back to `true` on its first line.

- [src/components/editor/Editor.jsx](src/components/editor/Editor.jsx) — `isSettingContentRef.current = false` added to effect cleanup

## Decisions made

- **Decision:** Retry loop in `main.js`, not in `package.json` shell script
  - **Why:** The bash subshell spawned by `concurrently` doesn't reliably inherit `node_modules/.bin` in PATH. JS retry logic with exact error-code matching (`ERR_CONNECTION_REFUSED`) is more portable and easier to reason about.
  - **Alternatives considered:** `wait-on` npm package before `electron .`; bash `until nc -z` loop; `sleep 2 && electron .` hardcoded delay

- **Decision:** Ref-mirror pattern for `suggestedFixesRef` (same as `latestIssuesRef`)
  - **Why:** `scheduleHideTooltip` must have a stable identity (empty deps) because it's called from mouse event handlers. Reading a ref is the standard way to give a stable callback access to current state without adding it to deps.

## Gotchas learned

- `concurrently` shells out with a minimal PATH; `node_modules/.bin` binaries like `electron` won't resolve in a bash subshell spawned from within it. Move all timing/retry logic to JS inside the process itself.
- On macOS, new `BrowserWindow` instances created by the main process can open behind already-focused windows. Always call `focus()` after `loadURL` on any programmatically-created window the user must interact with.
- The "cancelled" flag pattern for aborting stale async work does **not** automatically reset any ref that was set before the promise started. Any owning gate-ref must also be cleared in the cleanup function.
