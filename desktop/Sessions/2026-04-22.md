## What changed

### 1. gdoc initial lint race fix

**Root cause:** `markdownToHtml` for a large gdoc can resolve in <300ms (unified is near-synchronous as a microtask). But the 300ms lint debounce also fires and, if the DOM is still empty, falls back to raw markdown text for offset computation. When `onContentReady` fires (after `innerHTML = clean`), it was only re-applying stale wrong-offset issues rather than re-running lint.

**Fix:** Extracted lint body into a stable `runLintNow` `useCallback` in App.jsx. After setting `innerHTML`, Editor calls `onContentReadyRef.current?.()` which fires `runLintNow` immediately against the now-populated DOM — bypassing the debounce and getting correct plain-text offsets.

- [src/App.jsx](src/App.jsx) — added `useCallback` import; extracted `runLintNow`; lint `useEffect` now calls `setTimeout(runLintNow, 300)`; `onContentReady={runLintNow}` passed to `<Editor>`
- [src/components/editor/Editor.jsx](src/components/editor/Editor.jsx) — added `onContentReady` prop; added `onContentReadyRef` (always-fresh ref pattern); calls `onContentReadyRef.current?.()` after `innerHTML = clean` in content effect

### 2. Settings dialog body collapse

**Root cause:** `.modal-body { flex: 1 1 0 }` — flex-basis 0 in an auto-height flex container contributes nothing to intrinsic height, so the parent sizes to header+footer only and the body gets zero free space to grow into.

**Fix:** Changed to `flex: 1 1 auto` so the body's natural content height is used for the parent's intrinsic sizing; `flex-shrink: 1` + `min-height: 0` + `overflow-y: auto` then handles the scroll case when the card hits `max-height`.

- [src/styles.css](src/styles.css) — `.modal-body`: `flex: 1 1 0` → `flex: 1 1 auto`

## Decisions made

- **Decision:** `runLintNow` as a `useCallback` rather than inlining it in the `useEffect`
  - **Why:** Makes it callable from Editor's `onContentReady` without re-entering the debounce. The alternative (a `forceLintRef` updated in render) would need extra bookkeeping and wouldn't compose as cleanly with the deps system.
  - **Alternatives considered:** `contentReadyTick` state counter that the lint effect depends on; direct `setIssues` call from Editor (breaks the architecture boundary)

- **Decision:** `onContentReadyRef` pattern (always-fresh callback ref in Editor)
  - **Why:** The async `markdownToHtml` `.then()` callback closes over the prop value from when the effect was set up. Without a ref, it would call a stale `onContentReady` if the prop changed (e.g., `speData` loaded between the content effect running and the Promise resolving). The ref is updated via `useEffect([onContentReady])` which fires before the content effect on the same render.

- **Decision:** `flex: 1 1 auto` not `flex: 1 1 0` for scrollable modal body
  - **Why:** `flex-basis: 0` requires an explicit `height` on the parent for `flex-grow` to have free space to distribute. `flex-basis: auto` lets the body contribute its intrinsic content height, which works correctly with `max-height`-only constraints.

## Gotchas learned

- `flex: 1 1 0` is the CSS shorthand trap: it looks like "fill available space" but requires the parent to have an explicit height. `flex: 1 1 auto` is almost always what you want inside a `max-height`-constrained container.
- Electron's `app.getAppPath()` returns the `desktop/` dir in dev mode. The SPE defaults path walks `../resources/spe_defaults` from there — test confirmed in `main.spePath.test.js`.
- `useCallback` deps must exactly match what the callback closes over; having both the raw deps AND `runLintNow` in the `useEffect` dep array is redundant but not harmful.
